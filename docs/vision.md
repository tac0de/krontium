# Krontium Vision

## Current Web Positioning

Krontium.com/GitHub Pages is now an experimental CSS art site. The novel content is not the website's responsibility unless explicitly added later. The website should operate as a browser-native visual artwork that uses the name Krontium as a signal for mythic biology, pressure, and language, not as a conventional synopsis or lore archive.

## Source Motifs

The artwork may still draw from the novel's source motifs:

- Earth god: a tree-human form shaped by roots, bark, fungi, early land ecology, and deep soil memory.
- Sky god: a bird-human form shaped by feathered anatomy, flight, bone structure, and traces of dinosaur-to-bird evolution.
- Sea god: an octopus-human form shaped by cephalopod intelligence, chromatophores, soft anatomy, and deep marine life.

These motifs should appear as abstract systems: rings, tendrils, feather ribs, ink fields, pressure lines, typographic fragments, and responsive motion.

## Tone

Krontium should feel like a strange page the browser is barely containing: precise, tactile, multilingual, and non-template.

Use:

- pure CSS composition
- procedural-feeling geometry
- layered type
- controlled motion
- abstract biological forms
- black, bone, oxidized green, arterial red, deep-water blue, and bronze accents

Avoid:

- conventional novel marketing
- lore encyclopedia structure
- generic cards
- stock fantasy atmosphere
- overexplaining the story
- UI chrome that weakens the artwork

## Website Experience

The first screen is the artwork. The user should not feel they arrived at a blog, book promo, SaaS landing page, or template. Language routes may exist as alternate captions/labels, but the site itself is primarily visual.

Primary actions:

- inspect the moving composition
- switch language labels
- enter the experimental field
- follow minimal fragments if desired
