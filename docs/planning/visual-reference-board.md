# Krontium Visual Reference Board

Status: planning source for the next CSS art reset.

## Purpose

The current CSS diorama failed because it tried to draw literal god characters
without a mature visual grammar. The next work should not begin with character
parts. It should begin with a stable style system that can make simple shapes
look intentional rather than childish.

## Direction

Use a hybrid grammar:

- ancient rock art for mythic authority and deliberately primitive silhouettes
- biological scientific plates for structure, anatomy, and evolutionary logic
- portolan/map grammar for territory, routes, pressure, and human expansion

The result should feel like a discovered visual record, not a cartoon scene.

## Reference Set

## Reference: Lascaux cave painting system

- Source: https://www.metmuseum.org/essays/lascaux-ca-15000-b-c
- Domain: Paleolithic cave painting
- Composition: large animal forms dominate; humans are rare and visually minor.
- Surface: mineral pigment on cave wall, uneven ground, age and darkness.
- Typography: none.
- Color: ochre, charcoal, brown, black, mineral red.
- Information grammar: animal bodies, sparse human marks, signs, wall texture.
- Borrow: scale hierarchy where nonhuman powers dominate and humans are small.
- Avoid: copying specific animals or making fake archaeology.
- Candidate CSS translation: large god-sign silhouettes as stained or incised
  wall marks; tiny human fire marks as the only sharp animated points.

## Reference: Lascaux iconography categories

- Source: https://archeologie.culture.gouv.fr/lascaux/en/themes-0
- Domain: archaeological visual taxonomy
- Composition: animals, human figures, and signs are the core categories.
- Surface: cave wall as a containing field, not a modern flat canvas.
- Typography: none.
- Color: restrained mineral range.
- Information grammar: no landscape depiction; signs carry compressed meaning.
- Borrow: reduce Krontium to gods, humans, and signs rather than scenery.
- Avoid: literal landscape illustration as the main approach.
- Candidate CSS translation: earth, sky, sea gods as sign systems; monastery and
  human settlement as small repeated marks.

## Reference: Chauvet Panel of the Horses

- Source: https://smarthistory.org/theme-religion/
- Domain: prehistoric wall composition
- Composition: repeated animal figures produce directional momentum.
- Surface: dark wall, charcoal-like marks, partial overlap.
- Typography: none.
- Color: black, sepia, brown.
- Information grammar: repeated contours imply motion without animation.
- Borrow: overlap, partial bodies, directional charge.
- Avoid: isolated sticker-like figures.
- Candidate CSS translation: repeated wing ribs, root ribs, and tentacle contours
  should overlap into a frieze rather than stand as separate mascot bodies.

## Reference: Petroglyph and pictograph grammar

- Source: https://www.ancientartarchive.org/rock-art-what-are-pictographs-petroglyphs-and-geoglyphs/
- Domain: rock art classification
- Composition: human figures, animals, and symbolic motifs on stone.
- Surface: pigment decay, incised stone, incomplete marks.
- Typography: none.
- Color: ochre, charcoal, clay, surviving red marks.
- Information grammar: simplified but culturally weighted symbols.
- Borrow: intentional erosion, incomplete edges, incised groove shadows.
- Avoid: clean vector icon neatness.
- Candidate CSS translation: masks, rough borders, noise overlays, and clipped
  silhouettes that look worn into a surface.

## Reference: Haeckel marine biological plates

- Source: https://ocean.si.edu/ocean-life/invertebrates/art-forms-nature-marine-species-ernst-haeckel
- Domain: scientific-naturalist illustration
- Composition: specimens arranged with order and morphological comparison.
- Surface: printed plate, ink and color lithograph feel.
- Typography: usually labels in originals, but Krontium should suppress text.
- Color: natural but heightened; structured contrast.
- Information grammar: morphology, symmetry, repeated forms, taxonomy.
- Borrow: the sea god should become a cephalopod/anemone/radiolarian structure,
  not a round mascot head.
- Avoid: decorative prettiness without biological logic.
- Candidate CSS translation: radial membranes, cellular rings, chromatophore
  dot fields, tentacle cross sections.

## Reference: Haeckel radiolaria

- Source: https://publicdomainreview.org/collection/ernst-haeckel-s-radiolaria-1862
- Domain: microscopic marine morphology
- Composition: many small structures arranged as exact specimens.
- Surface: scientific print plate.
- Typography: original labels exist, but visual organization can survive without
  visible text.
- Color: linework-first.
- Information grammar: central capsules, skeletal geometry, radial repetition.
- Borrow: precise cellular geometry for the sea and early life layer.
- Avoid: generic circles.
- Candidate CSS translation: nested masks, repeating radial gradients, dotted
  skeletons, and specimen tray grids.

## Reference: Haeckel bats and wing anatomy

- Source: https://publicdomainreview.org/collection/ernst-haeckel-s-bats-1904/
- Domain: natural history plate
- Composition: wings treated as anatomical structures, not mascot symbols.
- Surface: printed plate.
- Typography: original labels secondary to form.
- Color: warm plate tones.
- Information grammar: bone span, membrane tension, comparative silhouettes.
- Borrow: sky god as anatomical wing spread with bone and feather logic.
- Avoid: simple triangle bird shape.
- Candidate CSS translation: ribbed spans, joint nodes, membrane masks, repeated
  feather-bone segments.

## Reference: Haeckel jellyfish

- Source: https://publicdomainreview.org/collection/ernst-haeckels-jellyfish/
- Domain: marine organism illustration
- Composition: translucent soft-bodied forms with radial order.
- Surface: delicate scientific illustration.
- Typography: not needed for Krontium.
- Color: restrained luminosity, fragile edges.
- Information grammar: soft anatomy, umbrellas, trailing structures.
- Borrow: sea god pressure and soft anatomy can be elegant without becoming a
  cartoon octopus.
- Avoid: cute eyes and round head.
- Candidate CSS translation: translucent layered masks, trailing filament lines,
  and low-opacity pressure bands.

## Reference: Portolan chart grammar

- Source: https://guides.loc.gov/nautical-charts/portolan-charts
- Domain: historical nautical chart
- Composition: coastline, compass rose, radiating rhumb lines, route logic.
- Surface: parchment chart.
- Typography: dense labels in originals, omitted or abstracted for Krontium.
- Color: parchment, black/red/blue linework.
- Information grammar: navigation, coastlines, bearings, scale.
- Borrow: the site can show territories and forces without drawing a literal
  fantasy map.
- Avoid: readable labels and decorative fantasy-map cliches.
- Candidate CSS translation: rhumb-line pressure fields, coastline-like god
  boundaries, route dashes from human settlement.

## Reference: Portolan construction method

- Source: https://www.cartographyunchained.com/chpo1/
- Domain: map construction analysis
- Composition: wind rose and projected directional lines.
- Surface: chart logic rather than illustration.
- Typography: not necessary.
- Color: line color can encode force direction.
- Information grammar: direction systems are functional, not decorative.
- Borrow: sky god can be expressed through wind direction systems rather than a
  bird mascot.
- Avoid: using compass roses as mere ornament.
- Candidate CSS translation: directional line networks that visibly govern sky,
  sea routes, and human expansion.

## Reference: Library of Congress Luso-Hispanic map guide

- Source: https://guides.loc.gov/luso-hispanic-world-in-maps/introduction/scope-of-this-guide
- Domain: historical map collection guide
- Composition: maps combine rhumb lines, wind roses, cartouches, and coast data.
- Surface: archival map.
- Typography: labels are common, but Krontium can abstract them.
- Color: aged substrate with controlled accent marks.
- Information grammar: authority comes from layered technical marks.
- Borrow: make the surface look like a working artifact, not a poster.
- Avoid: decorative map texture without systematic information.
- Candidate CSS translation: map substrate + god territories + route marks +
  human growth marks.

## Reference: Portolan chart visual density

- Source: https://mdl.library.utoronto.ca/mdl-blog/map-month-curious-world-portolan-charts
- Domain: nautical chart interpretation
- Composition: detailed coastlines plus rhumb-line network from wind roses.
- Surface: parchment, ink, dense navigational information.
- Typography: dense port labels in originals; suppressed here.
- Color: red/black wind directions, muted substrate.
- Information grammar: high density becomes professional when it follows rules.
- Borrow: use density only where it encodes territory, wind, sea, or human route.
- Avoid: random line clutter.
- Candidate CSS translation: controlled CSS line networks with different roles:
  wind, roots, currents, roads.

## Chosen Grammar

The next CSS artwork should be a no-text archaeological/specimen map:

- wall or parchment substrate
- three god systems as large nonhuman marks, not character mascots
- human civilization as tiny but precise fires, road marks, fences, tools, and
  route growth
- monastery as a central geometric neutral seal
- motion limited to pressure, ember, current, and growth signals

## CSS Art Rules

- Use `clip-path` for eroded silhouettes, not clean icons.
- Use pseudo-elements for incision shadows and pigment halos.
- Use repeating gradients for bark, feather ribs, coastline hatch, and sea
  pressure.
- Use layered masks for incomplete archaeological edges.
- Use small repeated marks for human advancement instead of drawing tiny people.
- Avoid eyes unless the whole grammar supports them; eyes made the current sea
  god childish.
- Keep all visible text at zero.
- Keep animation slow and environmental; no mascot bobbing.

## First Variant Batch

Generate variants across these axes:

- Surface: cave wall / parchment plate / basalt slab
- Composition: wall frieze / specimen plate / territorial map
- God expression: silhouette / anatomy / territory boundary
- Human expression: fire cluster / route network / fortification marks
- Monastery expression: seal / stair glyph / central void

Minimum batch: 30 variants.

Reject any variant that reads as:

- mascot
- sticker sheet
- generic fantasy map
- abstract background
- cute character scene

## Production Implication

Do not keep polishing the current `KrontiumDiorama` as-is. It should be treated
as a failed prototype. The next implementation should replace it with a
reference-led visual system.
